Необходимо реализовать и сравнить две версии системы обработки задач: синхронную (без брокера) и асинхронную (с брокером сообщений), а также провести простые замеры производительности.

Реализуйте API на FastAPI с endpoint POST /process, который выполняет “тяжелую” операцию (например, sleep(1–2 секунды)). В этой версии обработка должна происходить синхронно: запрос → обработка → ответ.

Переделайте систему с использованием брокера сообщений (RabbitMQ / Redis / Kafka — на выбор). Теперь логика:
FastAPI принимает запрос

сразу возвращает ответ ({"status": "accepted"})

Важно:
Разница обычно становится заметной начиная с 50–100 запросов или при параллельной нагрузке. Если вы не видите разницы — увеличьте количество запросов или степень параллелизма.

Цель задания — увидеть, как система начинает “ломаться” при синхронной обработке и как брокер помогает справляться с нагрузкой.
